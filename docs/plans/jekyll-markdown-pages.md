# Plugin Spec: `jekyll-markdown-pages`

## Goal

Create a Jekyll plugin (`source/_plugins/markdown_pages.rb`) that generates a clean markdown (`.md`) copy of every HTML page alongside its rendered output, following the [llms.txt spec](https://llmstxt.org/) recommendation.

## What it should do

For every page, post, and collection document that Jekyll renders to HTML, also write a `.md` file at the same path with `index.html.md` appended. Examples:

| HTML output | Markdown output |
|---|---|
| `/powerbeat/index.html` | `/powerbeat/index.html.md` |
| `/products/index.html` | `/products/index.html.md` |
| `/index.html` | `/index.html.md` |
| `/2024/some-post/index.html` | `/2024/some-post/index.html.md` |

## How it should work

1. Use a Jekyll hook on `:documents, :post_write` and `:pages, :post_write` (after HTML is written to disk).
2. For each document/page:
   - Skip if the output extension is not `.html`
   - Skip if the document has `markdown_copy: false` in front matter
   - Skip if it's a redirect page (has `redirect_to` in front matter, or generated by `jekyll-redirect-from`)
   - Recover the **raw markdown source** (the body after front matter, before any Liquid/layout rendering). Since `:post_write` runs after Jekyll has already rendered `document.content`/`page.content` to HTML, the plugin **must** re-read the source file from `document.path`/`page.path` and strip front matter using `Jekyll::Document::YAML_FRONT_MATTER_REGEXP` (or equivalent regex). Do **not** use `document.content` or `page.content` at this hook — they contain rendered HTML.
   - If the source markdown body does not already begin with an H1 (a line starting with `# ` after trimming leading whitespace/newlines), prepend a YAML-less header: `# {page.title}` followed by a blank line
   - If a page description is present in front matter, prepend it as a blockquote after the title/header block
   - Write this to `{destination_path}.md` (e.g., if the HTML is at `public/powerbeat/index.html`, write `public/powerbeat/index.html.md`)
3. Strip Liquid tags (`{% ... %}` and `{{ ... }}`) **only outside fenced code blocks** (` ``` ` or `~~~` with optional language identifier), since pages may intentionally document Liquid/Jekyll syntax inside code fences. Strip `{% raw %}...{% endraw %}` wrappers while preserving their inner content. Content inside fenced code blocks must be preserved verbatim.
4. Strip inline HTML/Jekyll-specific markup like `{:width="300" style="..."}` kramdown attribute annotations **only outside fenced code blocks**.

## Technical context

- **Jekyll version:** 4.4.1
- **Ruby version:** 3.2.2
- **Source dir:** `./source` (configured in `_config.yml`)
- **Destination dir:** `./public`
- **Collections:** `pages`, `assets`, `products`, `profiles` (all with `output: true`)
- **Existing custom plugin pattern:** See `source/_plugins/feed_generator_override.rb` for the project's plugin style:
  ```ruby
  # frozen_string_literal: true
  module JekyllFeed
    class Generator < Jekyll::Generator
      safe true
      priority :lowest
      def generate(_site)
      end
    end
  end
  ```
- **Layouts:** `base`, `home`, `page`, `post`, `product` — the plugin should NOT include layout HTML, only the page's own markdown content.

## Configuration

Support an optional `_config.yml` setting to disable the plugin:

```yaml
markdown_pages:
  enabled: true  # default true
  exclude:       # optional list of paths to skip
    - /privacy/
    - /error/
```

## Edge cases to handle

- Pages whose source content uses Liquid tags (e.g., `{% for product in site.products %}`) — these should be stripped, resulting in just the static markdown portions
- Pages with no meaningful markdown body (e.g., the products listing page is mostly Liquid) — still generate the `.md` file, but it may be mostly empty (just the title)
- Collection documents in `_assets/` — skip these (they're CSS, icons, txt files, not content pages)
- The `llms.txt` and `llms-full.txt` files — skip these (they're already markdown)

## Output format for each `.md` file

```markdown
# Page Title

> Page description (from front matter, if present)

{original markdown content with Liquid tags stripped}
```

## Testing

After implementing, run `bundle exec jekyll build` and verify:
- `public/powerbeat/index.html.md` exists and contains clean markdown
- `public/powernight/index.html.md` exists
- `public/products/index.html.md` exists
- `public/index.html.md` exists
- No `.md` file for redirect pages
- No `.md` file for assets collection items
